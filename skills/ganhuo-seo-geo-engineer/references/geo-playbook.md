# GEO Rebuild Playbook

## Rebuild Order

1. Capture the original meaning.
2. Extract core claims.
3. Build the question map.
4. Identify evidence-bearing material.
5. Add answer-first structure.
6. Convert loose paragraphs into reusable blocks.
7. Add FAQ, comparison, steps, limitations, and backlog.
8. Re-score and explain the delta.

## Claim Inventory

For every important claim, record:

| Field | Meaning |
| --- | --- |
| Claim | The statement being made |
| Current support | What the supplied material already contains |
| Gap | What is missing |
| Action | Keep, soften, rebuild, backlog, or remove |

## Preferred Structure

Use this structure for most standard rebuilds:

```markdown
# Title

## Core Answer
## Who This Is For
## Key Takeaways
## Main Content
## Practical Steps
## FAQ
## Limitations
## Backlog
```

## Depth Rules

- `light`: add answer block, clean headings, add 3-5 FAQ items, keep most paragraphs.
- `standard`: rebuild the article into answer-first sections and return the full package.
- `deep`: add tables, comparison blocks, multi-channel reuse, and detailed backlog.

## Review

Before final output, check:

- original meaning preserved;
- unsupported additions are not stated as facts;
- practical steps are concrete;
- FAQ answers real questions;
- limitations are visible;
- no keyword stuffing.
