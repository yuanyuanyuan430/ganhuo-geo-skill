# GEO Output Standard

## Required Package

Return the package in this order unless the user asks otherwise.

```markdown
# Rewritten Content

## GEO Scorecard

## Change Notes

## Supplement Backlog

## Risk Notes
```

## Scorecard

Use a compact table:

| Dimension | Before | After | Note |
| --- | ---: | ---: | --- |
| Evidence |  |  |  |
| Structure |  |  |  |
| Answerability |  |  |  |
| Entity clarity |  |  |  |
| Semantic coverage |  |  |  |
| Trust signals |  |  |  |
| Terminology |  |  |  |
| Robustness |  |  |  |
| Readability |  |  |  |

Score conservatively. A low score with a clear fix is more useful than a high score with vague praise.

## Change Notes

Use bullets grouped by action:

- Structure changes
- Evidence handling
- FAQ and answer blocks
- Entity and terminology cleanup
- Removed or softened claims

## Supplement Backlog

Use this table:

| Gap | Recommended addition | Priority |
| --- | --- | --- |
| Missing metric | Add real number, sample, and time window | P1 |
| Missing example | Add anonymized case or product screenshot | P1 |

## Risk Notes

Always mention:

- any claim that still needs human review;
- any time-sensitive data;
- any privacy-sensitive material;
- any ranking, traffic, or citation claim that should be avoided.
